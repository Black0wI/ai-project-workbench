# Architecture Review

Use this skill before high-impact implementation or when reviewing an architecture change.

## Workflow

1. Read relevant specs and ADRs.
2. Map system boundaries and data flow.
3. Identify coupling, hidden state, failure modes, and security risks.
4. Check whether AI provider interactions are isolated and observable.
5. Recommend minimal changes before implementation.

## Output Format

- Findings ordered by severity
- Open questions
- Required ADR updates
- Suggested validation
