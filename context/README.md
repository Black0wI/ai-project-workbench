The repository docs remain the official source of truth. Context integrations are auxiliary and must be reviewed before their content is promoted into specs, ADRs, prompts, or project instructions.

## Rules

- Do not commit private notes, client data, secrets, credentials, or proprietary research.
- Keep personal knowledge work outside the public repository.
- Promote only reviewed, sanitized context into versioned project docs.
