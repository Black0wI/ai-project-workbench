# Context Review Checklist

Use this before accepting Obsidian or external context into official project files.

- [ ] Source context is allowed to be used.
- [ ] No secrets, tokens, credentials, or private keys are present.
- [ ] No private customer or personal data is present.
- [ ] Obsidian `[[wiki links]]` are converted or intentionally documented.
- [ ] Claims and research findings include sources where needed.
- [ ] Context promoted to specs or ADRs is concise and decision-ready.
- [ ] Prompt drafts include safety notes and eval expectations.
- [ ] Official files remain the source of truth after promotion.
