# Design Review Checklist

Use this before approving UI or design system changes.

- [ ] `DESIGN.md` was read before implementation.
- [ ] UI follows the design intent and product domain.
- [ ] Token values are reused instead of hardcoded one-offs.
- [ ] Token changes are intentional and documented.
- [ ] Loading, empty, error, success, and permission states are covered.
- [ ] AI-generated content is visually distinct where applicable.
- [ ] High-impact AI actions require human confirmation.
- [ ] Keyboard navigation and focus states are present.
- [ ] Layout works on mobile, tablet, and desktop widths.
- [ ] Text does not overlap or overflow fixed controls.
