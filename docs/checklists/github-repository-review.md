# GitHub Repository Review Checklist

Use this when setting up or reviewing repository governance.

- [ ] GitHub is the canonical remote.
- [ ] Default branch is `main`.
- [ ] PR template is present.
- [ ] Issue templates are present.
- [ ] Required checks are defined.
- [ ] Branch protection proposal is user-approved before applying.
- [ ] Secret scanning is enabled where available.
- [ ] Repository visibility is intentional.
- [ ] Release and tag strategy is documented if releases are used.
- [ ] GitHub Actions permissions are least privilege where workflows are present.
