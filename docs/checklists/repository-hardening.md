# Repository Hardening Checklist

Use this when converting the template into a concrete project.

- [ ] Select and document the application stack.
- [ ] Add stack-specific lint, test, typecheck, and build commands.
- [ ] Add dependency scanning appropriate to the ecosystem.
- [ ] Add secret scanning in the hosting platform.
- [ ] Add branch protection rules.
- [ ] Add required CI checks.
- [ ] Confirm GitHub is the canonical remote.
- [ ] Document release and tag strategy if releases are needed.
- [ ] Add license and ownership metadata.
- [ ] Add deployment documentation.
- [ ] Update `AGENTS.md` with project-specific commands and constraints.
- [ ] Remove unused template sections.
