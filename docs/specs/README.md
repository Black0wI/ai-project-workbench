# Specs

Specs define what should be built and how it will be accepted.

## Naming

```text
NNN-short-feature-name.md
```

Examples:

- `001-user-authentication.md`
- `002-ai-triage-agent.md`
- `003-cloudflare-dns-routing.md`

## Workflow

1. Copy `000-template.md`.
2. Fill goals, non-goals, requirements, risks, and acceptance criteria.
3. Link required ADRs.
4. Capture AWS, Cloudflare, GitHub, PostgreSQL, DESIGN.md, and AI implications when relevant.
5. Link the spec from the PR.

## Status Values

- Draft
- Ready
- In Progress
- Shipped
- Superseded
